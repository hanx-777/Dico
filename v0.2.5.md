## 摘要

低秩适配（LoRA）已成为大语言模型参数高效微调的主流范式，其可训练参数量主要取决于各目标模块的 rank 分配。尽管近年来涌现了多种自适应分配方法，但它们普遍面临一个尚未被解决的瓶颈问题：现有方法均以整个模块为评估粒度，忽视了同一模块内部不同更新方向的异质性，且无法检测不同模块之间功能相似、高度重叠的更新方向。为了解决这一问题，我们提出 **DiCo**（Direction Coverage LoRA Allocation），将 rank 分配的基本竞争单位从模块下沉至 rank-one direction atom。具体而言，DiCo 将分配过程解耦为三个核心机制：首先，在冻结基座模型上通过 streaming randomized sketch 近似提取各模块的主导 SVD 方向；其次，通过梯度正交投影为每个 atom 构造带符号的样本级响应 profile，并引入梯度冲突检测以识别样本间需求矛盾的方向；最后，在真实参数预算约束下执行冲突感知的 coverage greedy 策略，通过 soft-tail 平滑过渡机制补齐至目标预算区间。在 GSM8K 数学推理任务上，DiCo-Sparse 以仅 15.6% 的参数量即超越 uniform 基准；而在更为严格的等参数量（~10.9M）对比中，预算受控变体 DiCo-$\eta$ 达到了 78.62% 的 Exact Match，相较于 uniform-r4 基准的 77.18% 实现了 +1.44% 的显著绝对提升，充分验证了方向级分配能够有效释放深层网络的潜能。

## 1 引言

### 背景

大语言模型（LLM）的参数规模持续增长，使得全参数微调在存储、通信和计算成本上愈发昂贵。参数高效微调（PEFT）方法通过仅训练少量新增参数来适配下游任务，大幅降低了微调开销。其中，LoRA（Low-Rank Adaptation）因其工程简洁性和推理零额外延迟的优势，已成为最广泛采用的 PEFT 方案之一：它在每个目标线性层旁引入低秩增量矩阵 $BA$，训练完成后可直接与原权重合并。在 LoRA 框架下，每个模块的可训练参数量由其分配的 rank 决定，因此 **如何在固定参数预算下分配各模块的 rank**，直接关乎参数效率与任务性能。

### 现有 Rank 分配的局限

最常见的做法是对所有目标模块分配相同 rank（uniform allocation），然而不同层、不同模块对下游任务的贡献差异显著，uniform 策略不可避免地将参数预算浪费在非关键模块上。针对这一问题，一系列自适应 rank 分配方法在**训练过程中**动态调整 rank：AdaLoRA (Zhang et al., ICLR'23) 将 LoRA 矩阵重参数化为 SVD 形式并逐步裁剪低重要性奇异值；SoRA (Ding et al., EMNLP'24) 引入可学习门控与 $\ell_1$ 正则促使部分维度归零；HALoRA (AAAI'26) 在层级与模块级构建层次化重要性评估。这些方法在识别模块间重要性差异方面取得了显著进展。然而，我们注意到两个尚未被充分探索的维度。**其一，跨模块方向冗余**：现有方法独立评估每个模块的重要性，不检测不同模块中是否存在功能相似的更新方向——即使两个模块各自重要，若它们的主导更新方向在样本响应上高度重叠，同时给予高 rank 仍会造成预算浪费。**其二，分配时机**：在训练过程中动态调整 rank 需要与优化过程耦合，引入额外调度策略（如 AdaLoRA 的 cubic schedule、SoRA 的正则化系数退火），并增加超参数调优成本；而若能在训练前通过校准信号一次性确定 rank pattern，则训练过程可保持标准 LoRA 的完全不变。

### 关键观察与直觉

我们的研究始于一个简单的出发点：LoRA 的每个 rank 对应一个 rank-one 更新 $b_k a_k^\top$，而一阶 Taylor 展开下，任务损失对权重更新的一阶需求恰好也是一个矩阵——梯度与激活的外积。因此，对校准样本的一阶响应矩阵 $R_m$ 做 SVD，其主奇异方向自然指示了该模块上最需要的 rank-one 更新方向。我们将这些方向称为 **direction atoms**——它们不是训练后的结果，而是训练前就可通过校准获得的任务适配需求信号。

对 Qwen3-8B 上 GSM8K 校准数据的分析揭示了两个关键现象 [Figure 1 placeholder]：**（a）模块内方向异质性**——同一模块内，不同 atom 的谱能量和样本响应模式差异悬殊，前 1--2 个 atom 往往集中了 80% 以上的谱能量，而靠后的 atom 贡献微弱；**（b）跨模块方向冗余**——不同模块（甚至不同层）的某些 atoms 在校准样本上的响应 profile 高度相似（余弦相似度 > 0.8），即它们响应相同的样本模式。这种冗余在模块级视角下不可见——两个模块各自的总体重要性可能都很高，但它们内部的主导方向却指向相同的任务子模式。

这两个现象共同指向一个结论：rank 分配的合适粒度不是模块，而是模块内部的 direction atom。同时，由于 atoms 是在训练前从冻结模型的一阶响应中提取的，它们反映的是**任务对模型的适配需求**，而非训练过程中的瞬时状态——这赋予了校准阶段 rank 分配以合理性。

### 核心观点与方法概述

基于上述观察，我们提出核心观点：

$$
\textbf{LoRA rank allocation 的基本竞争单位不应只是模块，而应是模块内部的 rank-one direction atom。}
$$

据此，我们提出 **DiCo**（Direction Coverage LoRA Allocation），一种基于冲突感知方向覆盖的预训练阶段 rank 分配方法。DiCo 的流程分为三个阶段：**（i）方向提取**——在冻结模型上通过 streaming randomized sketch 收集校准信号，近似提取每个目标模块的 top-$K$ 个 SVD direction atoms，避免显式构造完整聚合响应矩阵 $R_m$；**（ii）冲突感知 Profile 构建**——通过梯度正交投影为每个 atom 计算带符号的样本级投影系数，保留样本支持/反对该方向的方向性信息，并引入梯度冲突检测以量化不同样本对同一方向需求的矛盾程度；**（iii）冲突感知 Coverage 分配**——在真实 LoRA 参数预算约束下，以 coverage greedy 策略逐步选择覆盖残差最大且冲突度最低的 atom，并通过 soft-tail 平滑过渡机制补齐至目标预算区间。

### 贡献

本文的主要贡献如下：

1. **Direction-level rank allocation 框架。** 我们将 LoRA rank 分配的基本单位从模块下沉至 rank-one direction atom，并设计 sample-response coverage greedy 机制，通过样本维度的 profile residual 惩罚跨模块冗余方向，实现非冗余的 direction-level 分配。

2. **冲突感知的有符号 Profile。** 我们用有符号投影系数 $\alpha_{i,m,k}$ 替换无符号响应强度，保留样本对方向的支持/反对信息，并引入梯度冲突检测识别样本间需求矛盾的方向并在分配中降权。

3. **Streaming randomized sketch 后端。** 我们引入基于 Halko-Martinsson-Tropp 框架的 streaming sketch 方案，仅维护低维投影 $R_m\Omega$ 和 $Q_m^\top R_m$，避免显式构造 full response matrix $R_m$，将校准阶段的内存开销从数十 GB 降至 MB 量级。

4. **预算受控的冲突感知 Coverage 分配。** 我们在 coverage greedy 中同时考虑方向价值、样本覆盖、参数成本和梯度冲突，并通过 soft-tail 平滑过渡机制将最终 rank pattern 的真实 LoRA 参数量约束在 $[\eta \cdot B_{\text{ref}},\; B_{\text{ref}}]$ 区间内，确保与 uniform baseline 的公平比较。

5. **实验验证。** 我们在 GSM8K 数学推理任务上验证 DiCo 的有效性，参数效率变体 DiCo-Sparse 以极低参数量超越 uniform 基线；预算受控变体 DiCo-$\eta$ 在严格的等参数量条件下达到了 78.62% 的准确率（+1.44%），进一步验证了方向级分配的显著性能收益。

## 2 相关工作

### 2.1 参数高效微调

参数高效微调（Parameter-Efficient Fine-Tuning, PEFT）方法旨在冻结大部分预训练参数，仅训练少量新增参数以适配下游任务，从而大幅降低存储与计算开销。代表性方法包括：**Adapter** (Houlsby et al., 2019) 在 Transformer 各层中插入小型瓶颈模块；**Prefix Tuning** (Li & Liang, 2021) 在注意力层前添加可学习的连续前缀向量；**Prompt Tuning** (Lester et al., 2021) 在输入嵌入层前接可学习的 soft prompt；**LoRA** (Hu et al., 2022) 通过低秩矩阵分解 $W' = W + BA$ 表示权重更新，在推理时可与原权重合并，不引入额外延迟；**QLoRA** (Dettmers et al., 2023) 进一步结合量化技术，在 4-bit 基座模型上进行 LoRA 微调，进一步压缩显存占用。

在上述方法中，LoRA 因其工程简洁性、推理零开销和稳定的微调性能，已成为大语言模型 PEFT 的事实标准。本文在 LoRA 框架内研究 rank 分配问题，与上述方法互补而非替代。

### 2.2 自适应 LoRA Rank 分配

LoRA 的 rank 决定每个模块可学习的低秩方向数量。Uniform allocation 为所有模块分配相同 rank，忽略了模块间的贡献差异。近年来涌现了大量自适应 rank 方法，我们按其核心机制分为三类。

**训练时剪枝。** AdaLoRA (Zhang et al., ICLR'23) 将 LoRA 矩阵重参数化为 SVD 形式 $P \Lambda Q$，在训练过程中基于奇异值幅度与梯度的乘积定义重要性得分，逐步裁剪低重要性奇异值。值得注意的是，AdaLoRA 虽然操作奇异值维度，但其重要性得分是以整个模块的训练梯度累加而成，并通过 cubic budget schedule 全局调度——本质上仍是在模块级决定"保留多少奇异值"，而不分析各奇异方向在不同样本上的差异化响应。SoRA (Ding et al., EMNLP'24) 以更轻量的方式实现类似目标：在 LoRA 每个维度上引入可学习门控，通过 proximal gradient 优化促使部分维度归零。

**训练时扩展。** IncreLoRA (Zhang et al., 2024) 从低 rank 出发，根据梯度信号增量式扩展重要模块的 rank。AutoLoRA (Zhang et al., 2024) 采用元学习双层优化自动搜索各模块最优 rank。

**结构化分配。** HALoRA (AAAI'26) 构建层级与模块级的层次化重要性评估。IGU-LoRA (ICLR'26) 用 integrated gradients 替代瞬时梯度以获得更稳定的模块级重要性得分。

上述方法在模块间重要性差异建模上取得了显著成果。DiCo 与它们的核心区别在于三个维度：**(1) 跨模块冗余检测**——现有方法独立评估每个模块，不考虑不同模块是否存在功能重叠的更新方向；DiCo 通过样本-响应 profile 的 coverage residual 显式惩罚跨模块方向冗余。**(2) 信号来源**——训练时方法使用训练过程中的瞬时信号（梯度、门控值），随训练动态变化，且依赖调度策略控制剪枝/扩展节奏；DiCo 使用冻结模型上的一阶校准信号，反映任务适配的静态需求结构，与训练过程解耦。**(3) 分配时机**——DiCo 在训练前一次性输出 rank pattern，训练过程与标准 LoRA 完全一致，不引入额外训练开销。

### 2.3 随机化 SVD 与矩阵 Sketching

随机化 SVD (Halko, Martinsson & Tropp, 2011) 通过低维随机投影近似矩阵的主子空间，是大规模矩阵分解的标准工具。其核心思路是：对矩阵 $R \in \mathbb{R}^{m \times n}$，先计算 sketch $Y = R\Omega$（$\Omega \in \mathbb{R}^{n \times s}$ 为随机测试矩阵），再对 $Y$ 做正交化得到近似列空间基 $Q$，最终在低维子空间 $Q^\top R$ 中完成 SVD。该方法以 $O(mns)$ 的计算量替代 $O(mn \min(m,n))$ 的经典 SVD，且支持 streaming 计算——矩阵 $R$ 无需显式存储，只需逐块累加 sketch。

在 DiCo 中，randomized sketch 被用于近似提取校准信号矩阵 $R_m$ 的 top-$K$ 个 SVD direction atoms，而非近似模型权重本身。这一应用场景的关键区别在于：$R_m$ 是由所有校准样本的一阶响应逐样本累加而成的，天然适合 streaming sketch 的逐块累加模式。相比之下，现有 LoRA 方法中随机化技术主要用于权重初始化或压缩，与 DiCo 的校准信号提取目的根本不同。

## 3 预备知识与问题定义

### 3.1 LoRA 参数化

给定预训练模型中的一个目标线性层 $W_m \in \mathbb{R}^{d_{\text{out}} \times d_{\text{in}}}$，LoRA 通过引入低秩增量矩阵对其进行适配：

$$
W_m' = W_m + B_m A_m,
$$

其中 $B_m \in \mathbb{R}^{d_{\text{out}} \times r_m}$，$A_m \in \mathbb{R}^{r_m \times d_{\text{in}}}$，$r_m \ll \min(d_{\text{in}}, d_{\text{out}})$ 为模块 $m$ 的 rank。每增加一个 rank，模块 $m$ 的可训练参数增量为：

$$
c_m = d_{\text{in}} + d_{\text{out}}.
$$

### 3.2 Rank 分配问题

设 LoRA 目标模块集合为 $\mathcal{M}$，rank 分配向量为 $\mathbf{r} = (r_m)_{m \in \mathcal{M}}$。LoRA 的真实可训练参数量为：

$$
P(\mathbf{r}) = \sum_{m \in \mathcal{M}} c_m \, r_m.
$$

给定参数预算 $B$，rank 分配问题为：

$$
\max_{\mathbf{r}} \; \text{TaskPerformance}(f_\theta, \mathbf{r}) \quad \text{s.t.} \quad P(\mathbf{r}) \leq B.
$$

### 3.3 预算受控设置

给定 reference rank $r_{\text{ref}}$（例如 uniform-r4），reference budget 定义为：

$$
B_{\text{ref}} = r_{\text{ref}} \sum_{m \in \mathcal{M}} c_m.
$$

本文提出预算受控的 DiCo-$\eta$ 设置，要求最终 rank pattern 的真实参数量满足：

$$
\eta \cdot B_{\text{ref}} \leq P(\mathbf{r}) \leq B_{\text{ref}},
$$

其中 $\eta \in (0, 1]$ 为预算下限比例。默认设置 $\eta = 0.95$（DiCo-95），严格设置 $\eta = 0.98$（DiCo-98）。

我们强调，本文使用 **真实参数预算** $P(\mathbf{r})$ 而非平均 rank 作为公平比较标准。由于不同模块的 $c_m$ 不同（例如 attention projection 与 MLP projection 的维度差异），仅匹配平均 rank 并不等价于匹配可训练参数量，因此不适合作为预算公平性的度量。

## 4 方法：DiCo

DiCo 的核心思路可以用一句话概括：**提取每个模块的主导更新方向，通过有符号投影精确刻画其样本响应并识别梯度冲突，以冲突感知的 coverage greedy 选择最有价值且最不冗余的方向组合，直至用完预算。**

具体而言，给定冻结预训练模型 $f_\theta$、LoRA 目标模块集合 $\mathcal{M}$ 和校准数据集 $\mathcal{D}_{cal}$，DiCo 分三大步完成分配。**信号提取**（§4.1--4.2）：通过一阶响应的 SVD 提取每个模块的 direction atoms，并使用 streaming sketch 避免构造完整响应矩阵。**冲突感知 Profile 构建**（§4.3）：通过梯度正交投影为每个 atom 计算带符号的样本级投影系数，引入梯度冲突检测识别样本间需求矛盾的方向。**冲突感知 Coverage 分配**（§4.4）：在真实 LoRA 参数预算约束下，以 conflict-aware coverage greedy 策略选择最不冗余且冲突度最低的 atom，通过 soft-tail 平滑过渡补齐预算。图 2 展示了整体流程。

### 4.1 一阶响应与方向 Atoms

**样本级一阶响应。** 对于校准样本 $i$ 和目标模块 $m$，设 answer tokens 的位置集合为 $T_i$。在 teacher-forcing forward/backward pass 中，我们收集模块 $m$ 在每个 token 位置 $t$ 的输入激活 $a_{i,m,t}\in\mathbb{R}^{d_{in}}$ 和输出梯度 $g_{i,m,t}\in\mathbb{R}^{d_{out}}$。我们仅保留 answer tokens 的信号（通过 shifted label mask 排除 prompt tokens），使信号集中于模型的任务适配区域。样本级一阶响应矩阵定义为：

$$
G_{i,m}
=
\frac{1}{|T_i|}
\sum_{t\in T_i}
g_{i,m,t}\,a_{i,m,t}^{\top}
\in\mathbb{R}^{d_{out}\times d_{in}}.
$$

$G_{i,m}$ 刻画了样本 $i$ 在模块 $m$ 上对权重更新的一阶需求。

**聚合响应与 SVD 分解。** 跨样本聚合得到模块级响应矩阵：

$$
R_m = \sum_{i} G_{i,m}.
$$

对 $R_m$ 做截断 SVD：

$$
R_m \approx U_m \Sigma_m V_m^{\top},
$$

其中 $U_m\in\mathbb{R}^{d_{out}\times K}$，$V_m\in\mathbb{R}^{d_{in}\times K}$，$\Sigma_m=\operatorname{diag}(\sigma_{m,1},\dots,\sigma_{m,K})$。第 $k$ 个方向 atom 定义为：

$$
A_{m,k} = u_{m,k}\,v_{m,k}^{\top},
$$

其中 $u_{m,k}$ 和 $v_{m,k}$ 分别是 $U_m$ 和 $V_m$ 的第 $k$ 列。由于 LoRA 的每个 rank-one 更新同样具有外积形式 $b\,a^{\top}$，direction atom $A_{m,k}$ 与 LoRA rank 的几何单位完全对应。

**谱能量比。** 每个 atom 对聚合响应信号的解释占比为：

$$
\rho_{m,k}
=
\frac{\sigma_{m,k}^2}{\sum_{j=1}^{K}\sigma_{m,j}^2+\epsilon},
$$

其中 $\epsilon$ 为数值稳定常数。$\rho_{m,k}$ 在后续分配中作为 atom 重要性的先验权重。

### 4.2 Streaming Randomized Sketch

**动机。** 对于 Qwen3-8B 等大模型，32 层 × 7 模块的 fp32 完整 $R_m$ 总存储量约 30 GB，直接构造在内存和 PCIe 传输上均不可行。我们采用 streaming randomized sketch 近似提取 top-$K$ SVD atoms，核心思想是：**从不显式构造完整的 $R_m$ 或 $G_{i,m}$**。

**两步 sketch 流程。** 对每个模块 $m$，预先采样随机投影矩阵 $\Omega_m\in\mathbb{R}^{d_{in}\times s}$，其中 sketch 维度 $s=K+p$，$p$ 为 oversampling 参数。

*Range sketch。* 在每个校准样本的 forward/backward pass 中，以 streaming 方式累加：

$$
Y_m = R_m\,\Omega_m = \sum_{i} G_{i,m}\,\Omega_m.
$$

关键在于，$G_{i,m}\,\Omega_m$ 可分解为两次低维矩阵乘法而无需构造 $G_{i,m}$ 本身：

$$
G_{i,m}\,\Omega_m
=
\frac{1}{|T_i|}
\sum_{t\in T_i}
g_{i,m,t}\,(a_{i,m,t}^{\top}\Omega_m).
$$

其中 $a_{i,m,t}^{\top}\Omega_m\in\mathbb{R}^{1\times s}$ 是低维投影，整个计算在 $O(d_{out}\cdot s)$ 的空间复杂度内完成。

*正交化与低维 SVD。* 对 $Y_m$ 做 QR 分解获得正交基：

$$
Q_m = \operatorname{orth}(Y_m) \in\mathbb{R}^{d_{out}\times s}.
$$

通过第二轮 streaming pass 计算低维矩阵：

$$
B_m = Q_m^{\top} R_m = \sum_{i} Q_m^{\top} G_{i,m} \in\mathbb{R}^{s\times d_{in}}.
$$

同样，$Q_m^{\top} G_{i,m}$ 无需构造 $G_{i,m}$：

$$
Q_m^{\top} G_{i,m}
=
\frac{1}{|T_i|}
\sum_{t\in T_i}
(Q_m^{\top} g_{i,m,t})\,a_{i,m,t}^{\top}.
$$

**Atom 恢复。** 对小矩阵 $B_m\in\mathbb{R}^{s\times d_{in}}$ 做标准 SVD：

$$
B_m = \tilde{U}_m\,\Sigma_m\,V_m^{\top},
$$

恢复原始空间的左奇异向量：

$$
U_m = Q_m\,\tilde{U}_m.
$$

方向 atoms $A_{m,k}=u_{m,k}\,v_{m,k}^{\top}$ 和谱能量比 $\rho_{m,k}$ 均从该近似 SVD 中获得。整个流程仅维护 $O(d_{out}\cdot s)$ 和 $O(s\cdot d_{in})$ 量级的矩阵，存储开销较 full $R_m$ 降低约 $s/\min(d_{in},d_{out})$ 倍。

### 4.3 冲突感知的梯度分解 Profile

本小节是 DiCo v2 的第一个核心改进。传统方法仅以标量重要性分值排序模块；DiCo 进一步考察每个方向 atom 在不同校准样本上的响应模式。v2 通过梯度正交投影构建带符号的 profile，并引入梯度冲突检测，识别不同样本对同一方向需求矛盾的情况。

#### 4.3.1 有符号投影系数

对于 atom $(m,k)$，我们定义其在样本 $i$ 上的有符号投影系数：

$$
\alpha_{i,m,k} = u_{m,k}^{\top}\, G_{i,m}\, v_{m,k}.
$$

直觉上，$\alpha_{i,m,k}$ 是样本 $i$ 在模块 $m$ 的第 $k$ 个方向上的"投票"——正值表示该样本支持沿此方向更新权重，负值表示反对。与 v1 中的 sign-invariant profile $\psi_{m,k,i} = \|G_{i,m}^{\top}u_{m,k}\|_2 + \|G_{i,m}\,v_{m,k}\|_2$ 相比，有符号投影保留了方向性信息，能区分"所有样本温和支持"和"一半强烈支持、一半强烈反对"这两种截然不同的情况。

**Streaming 兼容性。** $\alpha_{i,m,k}$ 可分解为两个内积而无需构造 $G_{i,m}$：

$$
\alpha_{i,m,k} = \frac{1}{|T_i|} \sum_{t \in T_i} (u_{m,k}^{\top}\, g_{i,m,t})\,(a_{i,m,t}^{\top}\, v_{m,k}),
$$

计算量与原方法完全相同。

#### 4.3.2 梯度冲突检测

利用投影系数的符号分布，我们定义 atom $(m,k)$ 的梯度冲突度：

$$
\text{conflict}_{m,k} = 4 \cdot \frac{\#\{i : \alpha_{i,m,k} > 0\}}{N} \cdot \frac{\#\{i : \alpha_{i,m,k} < 0\}}{N}.
$$

该定义采用 $4p(1-p)$（即 Bernoulli 方差的 4 倍）形式，使得 conflict $\in [0, 1]$：当所有样本同向时 conflict $= 0$（完美一致），当正负各半时 conflict $= 1$（最大冲突）。

**冲突方向的语义。** 高冲突度意味着一半校准样本希望权重沿该方向更新，另一半希望沿相反方向更新。若为此方向分配 rank，LoRA 只能学到折中方案，两方面均难以充分受益。因此，高冲突方向在后续分配中被降权。

#### 4.3.3 Profile 归一化

Profile 向量改为基于有符号投影系数构建：

$$
p_{m,k} = (\alpha_{1,m,k},\, \alpha_{2,m,k},\, \dots,\, \alpha_{N,m,k})^{\top} \in \mathbb{R}^N.
$$

为消除不同校准样本之间梯度幅度的系统性差异，我们对 profile 进行两步归一化。

*样本级归一化。* 用样本在所有模块上的总响应强度作为归一化基准：

$$
\tilde{\alpha}_{i,m,k}
=
\frac{\alpha_{i,m,k}}
{\sum_{m'\in\mathcal{M}}\|G_{i,m'}\|_F+\epsilon}.
$$

*中心化与单位化。* 对每个 atom 的 profile 向量做中心化和 L2 归一化：

$$
\hat{p}_{m,k}
=
\frac{\tilde{p}_{m,k}-\operatorname{mean}(\tilde{p}_{m,k})}
{\|\tilde{p}_{m,k}-\operatorname{mean}(\tilde{p}_{m,k})\|_2+\epsilon},
$$

其中 $\tilde{p}_{m,k} = (\tilde{\alpha}_{1,m,k}, \dots, \tilde{\alpha}_{N,m,k})^{\top}$。归一化后的 profile $\hat{p}_{m,k} \in \mathbb{R}^N$ 表达的是 atom 在样本维度上的相对响应模式。由于保留了符号信息，两个 atom 的 profile 内积 $\hat{p}_a^{\top}\hat{p}_b$ 能更精确地刻画冗余度：正值表示被同一批样本正向支持（正冗余），负值表示一个的支持者是另一个的反对者（互补）。

### 4.4 冲突感知的 Coverage 分配

本小节将冲突感知 profile 与预算受控分配相结合。我们在 coverage greedy 框架中加入 conflict penalty，并用 soft-tail 平滑过渡替代原有的硬切换。

#### 4.4.1 Prefix 约束与 Coverage Residual

**Prefix 约束。** 由于同一模块的 SVD 方向按谱能量降序排列，我们要求 atom $(m,k)$ 仅在模块 $m$ 的当前 rank 恰好等于 $k$ 时才作为合法候选——即选择第 $k$ 个方向前，前 $k-1$ 个方向必须已被选中。这一约束尊重了谱排序的自然优先级。

**Coverage residual。** 设已选 atom profiles 张成的正交基为 $Q\in\mathbb{R}^{N\times d_Q}$，正交投影算子为 $P_Q=QQ^{\top}$。候选 atom $(m,k)$ 的 coverage residual 为：

$$
\operatorname{cov}_{m,k}
=
\|(I-P_Q)\,\hat{p}_{m,k}\|_2^2.
$$

该值越大，说明候选 atom 的样本响应模式越无法被已选 atoms 所解释，即其携带的信息越不冗余。由于 profile 现在是有符号的，coverage residual 能更精确地识别真正的冗余——同向支持的 atoms 被正确惩罚，而互补方向（一个的支持者是另一个的反对者）则保留较高的 coverage residual。

#### 4.4.2 模块重要性与冲突感知得分

**模块重要性。** 定义模块 $m$ 的重要性为样本响应 Frobenius 范数的几何均值：

$$
I_m = \exp\!\left(
\frac{1}{N}\sum_{i=1}^{N}\log(\epsilon+\|G_{i,m}\|_F)
\right).
$$

几何均值相较算术均值对异常样本更鲁棒。

**Conflict-aware coverage score。** 综合谱能量、模块重要性、coverage residual 和梯度冲突惩罚，候选 atom $(m,k)$ 的得分为：

$$
\operatorname{score}^{\text{cov}}_{m,k}
=
\frac{
I_m^{\beta}\;\rho_{m,k}^{\gamma}\;(1-\text{conflict}_{m,k})^{\delta}\;\operatorname{cov}_{m,k}
}{c_m},
$$

其中 $\beta,\gamma,\delta$ 为指数超参数，$c_m=d_m^{in}+d_m^{out}$ 为单 rank 参数成本。$(1-\text{conflict}_{m,k})^{\delta}$ 为梯度冲突惩罚项——高冲突方向被降权。分母 $c_m$ 使得不同维度的模块在参数效率意义下可比。

#### 4.4.3 Soft-Tail 平滑过渡

纯 coverage greedy 在覆盖信号饱和后自然停止，通常产生高度稀疏的 rank pattern（如 DiCo-Sparse 仅使用 Uniform-r4 约 15.6% 参数）。当需要与等预算 baseline 做公平比较时，需要继续补齐预算至目标区间。

**预算下限。** 定义最低预算要求：

$$
B_{\min} = \eta\,B_{\text{ref}},
$$

其中 $\eta\in\{0.95,0.98\}$ 对应 DiCo-95 和 DiCo-98。

**平滑过渡权重。** 我们用当前最大 coverage residual 的相对强度来控制 coverage 项与 spectral 项之间的平滑过渡：

$$
\lambda_{\text{cov}} = \min\!\left(1,\;\frac{\max_a\,\operatorname{cov}_a}{\epsilon_{\text{cov}}}\right),
$$

其中 $\epsilon_{\text{cov}}$ 为 coverage 饱和阈值。当 coverage 信号强时（$\max \operatorname{cov}_a \geq \epsilon_{\text{cov}}$），$\lambda_{\text{cov}} = 1$，得分完全由 coverage 主导；当 coverage 饱和（$\max \operatorname{cov}_a \to 0$），$\lambda_{\text{cov}} \to 0$，得分退化为冲突感知的谱排序。

**统一得分。** 融合 coverage 和 spectral 信号的统一得分为：

$$
\operatorname{score}_{m,k}
=
\frac{
I_m^{\beta}\;\rho_{m,k}^{\gamma}\;(1-\text{conflict}_{m,k})^{\delta}\;\bigl[\lambda_{\text{cov}}\cdot\operatorname{cov}_{m,k} + (1-\lambda_{\text{cov}})\bigr]
}{c_m}.
$$

该公式有三个优点：(1) coverage 还强时，主要按 coverage 选——与原 greedy 行为一致；(2) coverage 饱和时，自然退化为 conflict-aware spectral-tail 排序——即 $I_m^\beta \rho_{m,k}^\gamma (1-\text{conflict})^\delta / c_m$；(3) 不需要突然从 greedy 切到 tail，$\lambda_{\text{cov}}$ 提供连续平滑过渡，减少 $\epsilon_{\text{cov}}$ 的敏感性。

与原版 v0.2 的硬切换相比，soft-tail 保证了 tail 阶段也受 conflict penalty 调节——不再是"纯谱排序"，而是"冲突感知的谱排序"，这比原版更有原则性。

**终止条件。** 算法在以下任一条件满足时终止：

$$
P(\mathbf{r}) \geq B_{\min}
\quad\text{或}\quad\text{无合法候选},
$$

并在任何时刻保证 $P(\mathbf{r})\leq B_{\text{ref}}$。最终输出的 rank pattern 满足：

$$
\eta\,B_{\text{ref}} \leq P(\mathbf{r}) \leq B_{\text{ref}}.
$$

**Algorithm 1** 总结了完整流程。

#### 4.4.4 更一般的优化视角

从更一般的角度看，DiCo 的 coverage greedy 可以视为对一个 redundancy-regularized discrete allocation objective 的近似求解：

$$
\max_{\mathbf{x} \in \{0,1\}^{|\mathcal{A}|}} \;\; \sum_{a \in \mathcal{A}} \frac{w_a}{c_{m(a)}} \cdot x_a \;-\; \frac{\lambda}{2} \sum_{a \neq b} |\hat{p}_a^{\top} \hat{p}_b| \cdot x_a \cdot x_b,
$$

其中 $w_a = I_m^\beta \cdot \rho_{m,k}^\gamma \cdot (1 - \text{conflict})^\delta$ 为冲突感知的 atom 内在价值。Coverage greedy 可被理解为对该目标的贪心近似，其中 coverage residual 隐式地捕获了 pairwise 冗余项的边际贡献。未来可进一步探索 OT-style relaxation 或 potential-game interpretation，以获得更全局的分配策略。

---

**Algorithm 1: Sketch-Direction-DiCo**

**输入：** 校准数据 $\mathcal{D}_{cal}$，目标模块 $\mathcal{M}$，reference rank $r_{\text{ref}}$，预算下限比例 $\eta$，atom 数量 $K$，sketch 维度 $s$，coverage 阈值 $\epsilon_{\text{cov}}$，冲突惩罚指数 $\delta$

**输出：** Rank pattern $\mathbf{r}=(r_m)_{m\in\mathcal{M}}$

$\triangleright$ **Stage 1: Streaming Sketch**

1. 对每个模块 $m$：采样 $\Omega_m\in\mathbb{R}^{d_{in}\times s}$，初始化 $Y_m\leftarrow 0$
2. 对每个样本 $i\in\mathcal{D}_{cal}$：
   - 执行 teacher-forcing forward/backward
   - 对每个模块 $m$：收集 $\{(g_{i,m,t},\,a_{i,m,t})\}_{t\in T_i}$，累加 $Y_m\leftarrow Y_m+G_{i,m}\,\Omega_m$
3. 对每个模块 $m$：$Q_m\leftarrow\operatorname{orth}(Y_m)$，初始化 $B_m\leftarrow 0$
4. 对每个样本 $i\in\mathcal{D}_{cal}$（第二轮 pass）：
   - 对每个模块 $m$：累加 $B_m\leftarrow B_m+Q_m^{\top}G_{i,m}$

$\triangleright$ **Stage 2: Atom Extraction**

5. 对每个模块 $m$：$\tilde{U}_m,\Sigma_m,V_m\leftarrow\operatorname{SVD}(B_m)$；$U_m\leftarrow Q_m\tilde{U}_m$；保存 top-$K$ atoms 和 $\rho_{m,k}$

$\triangleright$ **Stage 3: 冲突感知 Profile 构建**

6. 对每个样本 $i$（第三轮 pass）：
   - 对每个 atom $(m,k)$：计算有符号投影系数 $\alpha_{i,m,k} = u_{m,k}^{\top} G_{i,m}\, v_{m,k}$
7. 对每个 atom $(m,k)$：计算 $\text{conflict}_{m,k} = 4 \cdot \frac{\#\{i:\alpha_{i,m,k}>0\}}{N} \cdot \frac{\#\{i:\alpha_{i,m,k}<0\}}{N}$
8. 归一化得到 $\hat{p}_{m,k}$（样本级归一化 → 中心化 → L2 单位化）

$\triangleright$ **Stage 4: Conflict-Aware Coverage Greedy + Soft-Tail**

9. 计算 $B_{\text{ref}}\leftarrow r_{\text{ref}}\sum_m c_m$，$B_{\min}\leftarrow\eta\,B_{\text{ref}}$
10. 初始化 $r_m\leftarrow 0\;\forall m$，正交基 $Q\leftarrow\emptyset$
11. **while** 存在合法候选 **and** $P(\mathbf{r})<B_{\text{ref}}$ **do**
    - 确定满足 prefix 约束且 $P(\mathbf{r})+c_m\leq B_{\text{ref}}$ 的候选集
    - 对每个候选计算 $\operatorname{cov}_{m,k}$
    - 计算 $\lambda_{\text{cov}}\leftarrow\min(1,\max_a \operatorname{cov}_a / \epsilon_{\text{cov}})$
    - 对每个候选计算统一得分 $\operatorname{score}_{m,k}$
    - 选择得分最高的 atom $(m^*,k^*)$
    - $r_{m^*}\leftarrow r_{m^*}+1$；Gram-Schmidt 更新正交基 $Q$
    - **if** $P(\mathbf{r})\geq B_{\min}$ **and** $\lambda_{\text{cov}} < 1$ **then break**
12. **return** $\mathbf{r}$

**冲突检测的作用。** 移除冲突惩罚后，高冲突方向不再被降权，可能导致 LoRA 在这些方向上学到的更新因样本间矛盾而效果减弱。

**有符号 Profile 的作用。** 回退到无符号 profile 后，coverage residual 丧失方向性信息，无法区分"正冗余"和"互补"。我们预期此变体的 rank pattern 在跨模块冗余识别上精度下降。

**Soft-tail vs Hard-tail。** 硬切换版本在 coverage residual 低于 $\epsilon_{\text{cov}}$ 后突然切换至纯谱排序，切换点处的得分不连续可能导致次优选择。Soft-tail 提供平滑过渡，减少对 $\epsilon_{\text{cov}}$ 的敏感性。

**Tail 阶段的 Conflict Penalty。** 原版 v0.2 的 tail 阶段仅按 $I_m^\beta \rho_{m,k}^\gamma / c_m$ 排序，不考虑冲突。加入 conflict penalty 后，tail 阶段也避免选择高冲突方向。

**Coverage 的作用。** 移除 coverage residual 后，分配退化为纯冲突感知的谱排序，无法识别跨模块方向冗余。我们预期此变体将产生更多集中于少数高重要性模块的 rank pattern。

**Sketch 维度。** $s=16$ 可能因近似精度不足导致 atom 质量下降；$s=64$ 增加计算成本但提供更高保真度。默认 $s=32$ 在精度与效率间取得平衡。

**预算下限 $\eta$。** 较低的 $\eta$（如 0.75）允许更稀疏的 rank pattern，更接近 DiCo-Sparse 行为；$\eta=0.98$ 则强制更充分地利用预算。

## 6 结论

本文提出 DiCo，一种将 LoRA rank 分配单位从模块扩展至 rank-one direction atoms 的预算受控方法。DiCo 通过 streaming randomized sketch 提取方向 atoms，利用有符号投影系数构建冲突感知的 profile 以精确刻画方向质量，并基于 conflict-aware coverage greedy 策略在 soft-tail 平滑过渡下选择最不冗余且冲突度最低的方向组合。实验表明，direction-level 选择不仅能在极低参数下保持高效率，更在同等参数量（10.91M）的严格对比中达到 78.62% EM，显著超越均匀分配的 77.18% EM，验证了方向级视角的有效性。

本方法目前在 calibration 阶段仍需多轮 forward/backward pass，且当前实验仅覆盖 GSM8K 单一任务。未来工作将沿三个方向展开：(1) 将 DiCo 扩展至多任务联合校准场景，探索跨任务方向冲突的建模；(2) 在更大规模模型（如 70B 级别）上验证 sketch 的可扩展性；(3) 探索将 coverage greedy 替换为 OT-style relaxation 或 potential-game interpretation，以获得更全局的分配策略。
